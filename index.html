Qamar Privacy Policy

Your Islamic Companion Discord Bot

Welcome to Qamar, your dedicated Islamic companion bot for Discord. We are committed to protecting your privacy and being transparent about how we handle your data. This privacy policy explains what information we collect, how we use it, and your rights.

About Qamar

Qamar is a Discord bot designed to enrich your Islamic experience with features including:

🕌 Prayer Times Accurate daily prayer schedules
📖 Quran Verses Random Quranic verses with translations
📿 Dhikr Counter Track your remembrance of Allah
⏰ Reminders Custom reminders for important tasks
🙏 Islamic Duas Random duas for various occasions
🧭 Qiblah Direction Direction towards the Kaaba

Information We Collect

User-Provided Information

  • Discord User ID - Used to identify and personalize your experience
  • Location - Stored only when you voluntarily set it using /setlocation for prayer time calculations
  • Dhikr Counter Data - Your personal dhikr tracking count (stored in memory during bot sessions)
  • Reminder Preferences - Reminders you set with the /remind command

Automatically Collected Information

  • Timestamps of bot interactions
  • Commands executed (for logging and improvement purposes)
  • Server/Guild IDs where the bot is active

Data Storage & Security

We take data security seriously. Here's how your information is handled:

Local Storage Format

User data is stored in a users.json file maintained on our secure servers. This file contains:

  • Your Discord User ID
  • Your configured location for prayer times

Important: This data is never shared, exported, or used for any purpose other than providing Qamar's services.

In-Memory Data

The following data is stored temporarily in the bot's memory and is lost when the bot restarts:

  • Dhikr Counter: Your SubhanAllah count resets when the bot restarts
  • Reminders: Active reminders are processed and removed after being sent

Encryption & Protection

  • Data is stored securely with appropriate access controls
  • Only the bot owner can access the users.json file
  • Regular backups ensure data integrity
  • No unauthorized third parties have access to your information

Third-Party APIs

Qamar integrates with the following external services to provide its features:

🕌 Aladhan API

Used for retrieving accurate prayer times based on your location.

  • Your location (when you set it) is sent to this API
  • Prayer time data is provided in real-time
  • No personal identifiers beyond the location are sent
  • API endpoint: http://api.aladhan.com/v1/timingsByCity

📖 AlQuran Cloud API

Provides authentic Quran verses with translations and audio recitations.

  • No personal data is shared with this service
  • Only requests for random Quranic verses are made
  • Includes Arabic text, English translations, and audio recitation links
  • API endpoints: https://api.alquran.cloud/v1/

Note: These are well-established Islamic APIs with their own privacy policies. We recommend reviewing them at their respective websites.

How We Use Your Data

  • Prayer Times: Display accurate prayer schedules based on your set location using the Aladhan API
  • Quran Verses: Retrieve and share random Quranic content when requested via /quran
  • Dhikr Counter: Track and maintain your personal dhikr statistics during bot sessions
  • Reminders: Send you notifications based on your configured reminders using /remind
  • Prayer Notifications: Notify you when prayer times are near using /pingwhensalah
  • Bot Improvement: Anonymized command usage helps us improve the bot's features

✓ We never sell, trade, or share your data with third parties
✓ Your data is used exclusively for providing Qamar's services
✓ We don't use your information for advertising or marketing
✓ We don't collect or store passwords, tokens, or sensitive credentials

Commands & Features

Available Commands

  • /salam - Say Salam to the bot (Wa Alaikum Assalam response)
  • /remind - Set a custom reminder with duration (e.g., 5m, 2h, 1d)
  • /randomdua - Get a random dua with Arabic and English translation
  • /qiblah - Get the qiblah direction towards the Kaaba
  • /dhikircounter - Start the interactive dhikr counter with SubhanAllah button
  • /setlocation - Set your location for prayer time calculations
  • /prayertimes - Get prayer times for your set location
  • /pingwhensalah - Get notified when the next prayer time is near
  • /quran - Get a random Quran verse with audio recitation
  • /help - Display all available commands
  • /invite - Get the bot's invite link
  • /ping - Check the bot's latency

Your Rights & Control

You have complete control over your data:

View Your Data

You can view your stored location and other settings by using the bot's configuration commands.

Delete Your Data

You can manually delete your location setting, which will remove your data from users.json. Contact the bot owner for complete data removal requests.

Update Your Information

You can update your location at any time using the /setlocation command with a new location.

Opt-Out of Features

Simply don't use the features you don't want. For example, if you don't want prayer time notifications, don't use /setlocation or /pingwhensalah.

Data Retention

Here's our data retention policy:

  • Location Data (users.json): Stored indefinitely until you remove it or request deletion
  • Dhikr Counter: Stored in memory during bot session; resets when bot restarts
  • Reminders: Deleted after being sent or completed
  • Command Logs: Stored temporarily for debugging and improvement
  • On Request: Contact the bot owner for data deletion requests

Children's Privacy

Qamar is not intended for children under 13 years of age. We do not knowingly collect information from children under 13. If we become aware that a child under 13 has provided us with personal information, we will immediately request deletion of such information.

Security Measures

We implement comprehensive security measures to protect your data:

  • Secure local storage with restricted file access
  • Discord API security best practices
  • No storage of sensitive data (passwords, tokens, etc.)
  • HTTPS/TLS encryption for all external API communications
  • Immediate response to any security incidents
